MdMd を .md の既定アプリにする

.md がすでに別のアプリ(メモ帳 / Typora / VS Code など)に 紐づけられている場合、Edge は自動関連付けダイアログを出さないことがあります。 そのときは下記いずれかの方法で MdMd に切り替えてください。

A. Edge から再登録 B. エクスプローラーから C. Windows 設定から うまく行かないとき
前提: install.cmd から PWA インストール(Edge 上で「インストール」ボタン) まで済んでいること。edge://apps に MdMd が並んでいれば OK。

A. Edge から関連付けダイアログを再表示

一番ラクなルート。Edge 自身に「もう一度関連付けを聞き直して」と指示する方法です。

  1. Edge のアドレスバーに edge://apps を入力
  2. 一覧から MdMd をクリック
  3. 「アプリの詳細」を開く
  4. 「サポートされている種類のファイルでこのアプリを使用するか確認する」 というトグルを オフ → オン に切り替える
  5. 切り替えた瞬間、Edge が確認バナーを出してきます → [許可] を押す
これで関連付け完了。 任意の .md をダブルクリックして MdMd が起動すれば OK。

B. エクスプローラーから手動関連付け

方法 A でダイアログが出ない、または既定アプリだけ強制的に書き換えたい場合。

  1. エクスプローラーで任意の .md ファイルを 右クリック
  2. 「プログラムから開く」 → 「別のアプリを選択」
  3. リストから MdMd を選ぶ
    リストに MdMd が出ない場合
    • 「PC でアプリを選択する」 / 「常にこのアプリで開く」を一旦タップせず、 そのままダイアログ下部の「その他のアプリ」を展開
    • それでも無ければ Edge を一度終了して、エクスプローラーを再起動してから再試行
    • 最終手段は 方法 C
  4. 下部の「常にこのアプリを使って .md ファイルを開く」に チェック → [OK]
  5. .markdown も同様に関連付け(.txt は要否で判断)

C. Windows 設定 → 既定のアプリ

方法 A・B が両方ダメなときの確実ルート。

  1. Win + I で「設定」を開く
  2. アプリ → 既定のアプリ
  3. 検索ボックスに MdMd と入力して選択
  4. 関連付け一覧に .md .markdown .txt が並ぶので、 既定値ボタンを押して MdMd を選ぶ
逆引きルート: 拡張子から既定アプリを設定したい
  1. 設定 → アプリ → 既定のアプリ → 「ファイルの種類で既定値を選択する」
  2. 一覧を下にスクロールして .md を探す
  3. 右の現在の既定アプリをクリック → MdMd を選ぶ

うまく行かないときのチェックリスト

① MdMd がアプリ一覧に出ない

  • PWA インストールが失敗している可能性 → install.cmd を再実行 → Edge で「インストール」 → edge://apps に並ぶか確認
  • Edge を完全終了(タスクトレイの常駐含む)してから再起動 → 再試行

② ダブルクリックしても何も起きない / 別アプリが開く

  • 関連付け側ではなく、ファイルの「開く方法」のロックが残っている可能性。 方法 B でチェックボックスを付けて再設定
  • VS Code 等が .md をシェル統合経由で予約していると、Windows の 関連付けより優先されることがある。該当アプリの設定で 「.md をこのアプリで開く」を外す

③ Edge のダイアログが「自動的に非表示」と出る

  • 「今後表示しない」を以前押した可能性。方法 A の アプリ詳細トグルから復活させる

④ 組織の管理 PC で「既定アプリの変更が無効」

グループポリシーで関連付けが固定されているケース。 このときは IT 部門に「.md を MdMd PWA に関連付けたい」と相談するしかない。 PWA 単体の起動・編集・保存は依然として動作するので、 ダブルクリック以外(D&D / Ctrl+O)経由で運用する手もあります。

⑤ 関連付けを完全リセットしてやり直したい

設定 → アプリ → 既定のアプリ → ページ下部の「リセット」ボタン → すべての既定値が初期化される → install.cmd 経由でやり直す

動作確認

  1. 任意の .md をダブルクリック
  2. MdMd の単独ウィンドウが開く
  3. その .md がタブとして自動で読み込まれる
  4. 編集 → Ctrl + S → 元の .md が上書きされる
全部通れば file_handlers + launchQueue + File System Access API が 正しく機能している証拠です。以後ネット完全切断状態でも動作します。
